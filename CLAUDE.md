# TypoFix — головне для кожного воркера

TypoFix — кросплатформний (Windows+macOS) аналог Punto Switcher: визначає текст,
набраний у неправильній розкладці, перемикає розкладку і **перенабирає** його.
Стек: **Rust** (ядро+платформа) + **Tauri** (трей+налаштування) + **Svelte** (UI).
Пріоритет №1 — **влучність розпізнавання**; precision важливіша за recall.
Повна документація: `docs/` (ARCHITECTURE, DEVELOPMENT_PLAN, TESTING_STRATEGY, CONCEPT).

## Залізні правила (неочевидне з коду — порушувати не можна)
1. **`typofix-core` — чистий і детермінований.** Жодних ОС-викликів, годинника,
   random. Час приходить у подіях (`timestamp_ms`). Уся логіка — `step(state, event, ctx) -> Vec<Action>`.
2. **Перенабір — лише через `Action::TypeUnicode(String)`** (готовий текст), НЕ повтор
   scancode у новій розкладці (інакше гонка з асинхронним перемиканням). `SwitchLayout`
   — окрема дія, потрібна тільки щоб *подальший* набір користувача був правильним.
3. **Інвалідація буфера = блокер втрати даних.** Per-window буфер скидається на
   клік миші / стрілки / Home-End / виділення / зміну фокуса / auto-repeat. У core
   йдуть УСІ `InputEvent`, не лише клавіші.
4. **Приватність:** буфер натисків лише в RAM, ніколи на диск. Password/secure-поля
   не буферимо. Golden-фікстури — ТІЛЬКИ синтетичні/скрабовані (нуль реальних секретів).
5. **Розкладки в рантаймі беремо з ОС** (`ToUnicodeEx`/`UCKeyTranslate`); TOML — лише
   fallback/еталон. AltGr/dead-keys — через ОС.
6. **Влучність:** поріг залежить від довжини слова; датасет має НЕГАТИВНИЙ клас
   (легітимний текст, який НЕ чіпати).

## Робочий процес
- Кожна фіча: unit-тест + integration-сценарій (через `typofix-platform-virtual`).
  Баг → спершу regression-тест, потім фікс. `clippy --all-targets -- -D warnings` і
  `cargo fmt` обов'язкові перед завершенням.
- **Автономія:** повна для core (virtual platform); платформні баги ловить лише
  real-OS харнес.
- **Git:** воркерам заборонено commit/push/stash/rebase/merge/reset (це робить
  оркестратор). `git status`/`diff`/`log` — вільно.

## Non-goals
IME-мови (CJK), елевовані вікна Windows (UIPI), поля паролів, орфографія, хмара.

## Правило про CLAUDE.md
Якщо в директорії/модулі є **неочевидне** правило чи готча (те, що не видно з коду
і важко самому дослідити) — додай/онови локальний `CLAUDE.md`. Тримай коротко;
лише найголовніше. Не дублюй те, що очевидне з коду.
