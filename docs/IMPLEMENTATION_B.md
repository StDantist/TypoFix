# TypoFix — План імплементації B1–B6 (інтерфейс і керування)

> Робочий playbook оркестратора для реалізації решти `docs/ROADMAP.md`
> (блок **B. Інтерфейс і керування**). A1/A2 — вже зроблено. Цей файл —
> як саме розпаралелити по воркерах, з контрактами між крейтами, щоб
> уникнути контенції в спільному дереві.

## Принципи (фільтр для кожної фічі — з ROADMAP)
- **Privacy-first:** усе локально, нуль телеметрії, буфер лише в RAM.
- **Precision > recall:** кожна нова евристика — з precision-гейтом + regression-тестом.
- **Легкість:** мала апка, трей, не навантажує систему.
- Кожна фіча: unit-тест (core) + integration (virtual) + ворота `clippy -D warnings`+`fmt`+`npm build`.

## Команда й володіння крейтами (один воркер на крейт — анти-контенція)
- **Den** — `typofix-core` (логіка: undo/revert, manual-switch, case-transform, нові toggle-поля DetectorConfig).
- **Pylyp** — `typofix-platform-windows` (selection-grab, layout cycle, real-OS smoke хоткеїв).
- **Bruno** — `typofix-data` (дані для мовних пар / звуковий асет, якщо треба).
- **Taras** — `src-tauri` + `ui/` (плагіни Tauri, конфіг-DTO, runtime-routing, всі UI-картки/i18n).
- **Knox** — резерв / другий UI-воркер у фазах, де Taras на бекенді (тільки на НЕ-перетинних файлах).

**Правило контенції:** `src-tauri`/`ui` — одночасно ОДИН воркер на КОЖЕН конкретний файл.
`App.svelte`, `lib.rs`, `config.rs`, `runtime.rs` — серіалізувати. Core/platform/data
йдуть паралельно з src-tauri вільно (різні крейти).

## Фаза B1 — Гарячі клавіші (НАЙВИЩИЙ пріоритет)

Лікують свідомий «недосил» (precision > recall): користувач виправляє пропуск/помилку однією клавішею.

### Підфічі
1. **Скасувати останнє перемикання** — повернути оригінал + завчити слово (не чіпати надалі).
2. **Перемкнути вручну** останнє слово / виділення (апка пропустила, а треба).
3. **Примусово змінити розкладку** останнього слова (аналог подвійного Shift) — варіант №2.
4. **Змінити регістр** виділеного (ВЕЛИКІ / малі / Як_Речення).
5. **Пауза / відновлення** однією клавішею.
6. Усі клавіші — перепризначувані, вимикані.

### Контракт API (щоб Den і Taras працювали ПАРАЛЕЛЬНО)
`typofix-core` надає чисті функції/методи (Den реалізує, Taras викликає з runtime):
- `revert_last(state: &mut EngineState) -> Vec<Action>` — інверсія останнього авто-перенабору
  (DeleteChars(новий_len) + TypeUnicode(оригінал) + SwitchLayout(назад)) + `learned.learn(word)`.
  Якщо нічого скасовувати — `vec![]`.
- `force_switch_last(state: &mut EngineState, ctx: &Context) -> Vec<Action>` — взяти останній
  word-буфер і перенабрати в іншій мові БЕЗ порогу (ручне рішення користувача завжди перемагає).
- `transform_case(text: &str, mode: CaseMode) -> String` — ЧИСТА; `CaseMode{Upper,Lower,Sentence}`.
  Працює над переданим текстом (виділення приходить ззовні, не з буфера).

Den: тести через `typofix-platform-virtual` (revert повертає рівно оригінал; force_switch ігнорує поріг; case-режими). Жодного IO/часу в core.

### Розподіл
- **Den (core):** три API вище + unit/integration тести. Старт одразу (контракт зафіксовано).
- **Taras (src-tauri+ui):**
  - `tauri-plugin-global-shortcut` у `Cargo.toml`+`tauri.conf.json`+`capabilities`.
  - `HotkeysDto` у `AppSettings` (рядки-акселератори на кожну дію, кожна вимикана), дефолти, persist.
  - `src-tauri/src/hotkeys.rs` — реєстрація/перереєстрація з конфіга; роутинг кожного хоткея в `RuntimeManager`.
  - Роутинг: pause→toggle `enabled`; revert/force-switch→виклик core-API через engine; case→clipboard-раунд-тріп (див. Pylyp).
  - UI-картка «Гарячі клавіші»: захоплення комбінації, вимкнення кожної, i18n.
  - **Почати з ПАУЗИ** (найпростіша, суто `enabled`-toggle) — швидка перемога, далі revert/force/case.
- **Pylyp (platform-windows):**
  - `get_selection_text()` — узяти виділений текст (синтет. Ctrl+C → читання буфера обміну, з відновленням попереднього вмісту) для case-зміни та «перемкнути виділення».
  - Real-OS smoke: хоткеї реально спрацьовують, revert/force-switch коректно перенабирають у фокусному вікні.
  - **НЕ чіпати `live_engine.rs`** (чужа pre-existing зміна).

### Гейти
Privacy: clipboard-вміст відновлювати після читання; нічого не логувати. Хоткеї дефолтно
розумні й не конфліктні; усі вимикані. Регресія: virtual-сценарій «revert повертає точний оригінал».

## Фаза B2 + B4 — Зворотний зв'язок + перемикачі поведінки (разом, обидва Taras-важкі)

### B2. Зворотний зв'язок
- **Звук** при перемиканні (дефолт ВИМКНЕНО, вимикається). Bundled wav / системний звук; емітиться з runtime на successful retype.
- **Індикатор у треї** — поточна розкладка; інша іконка на паузі (вже є toggle-стан у lib.rs).
- Опційно **коротка спливашка** біля курсора на мить після виправлення (вимикається) — окреме маленьке tauri-вікно без рамки.

### B4. Перемикачі поведінки (один людський рядок на кожне)
- **Виправляти регістр** (`ПРивіт→Привіт`) — on/off.
- **Forex-режим** (валютні пари) — on/off.
- **Розпізнавати файлові розширення** (A1) — on/off.
- **Фонотактика** (ь-на-початку) — on/off.
- **Чутливість** — людський повзунок «обережно ↔ агресивно» (мапиться на наш поріг; зробити зрозумілим).
- **Виправляти випадковий CapsLock**.

### Контракт (Den ↔ Taras)
- Den: переконатися, що `DetectorConfig` має РЕАЛЬНІ прапорці для case-fix і forex (а не always-on);
  додати/перевірити `case_fix_enabled`, `forex_enabled` (phonotactics/extensions вже є). CapsLock-евристику — core-правило за прапорцем + regression-тест.
- Taras: `BehaviorDto` у `AppSettings` → мапінг у `DetectorConfig` (`runtime.rs`), UI-картка тогглів + повзунок чутливості + i18n. B2: звук-опція в DTO, емісія в runtime, трей-іконки/індикатор у `lib.rs`.
- Bruno: підготувати маленький звуковий асет у `data/` (якщо не системний звук).

### Розподіл
Den (core toggles + CapsLock-правило) ‖ Taras (DTO+UI+трей+звук). Bruno: звук-асет.
Pylyp: real-OS перевірка трей-іконок/звуку (легка). Knox може взяти B2-sp
поп-ап-вікно (окремий tauri-window-файл, НЕ App.svelte) паралельно Taras-картці тогглів.

## Фаза B3 — Навчені слова + per-app вимкнення

- **Перегляд авто-навчених слів** (`learned_exceptions.txt`) — список у UI з кнопкою «прибрати помилкове».
  - Den/core: expose читання списку (вже є `LearnedExceptions`); Taras: tauri-команда `list_learned`/`remove_learned` + UI-модалка.
- **Опція «у цій програмі взагалі не працювати»** (повне per-app вимкнення) — DTO вже має exclusions; додати UI-режим «вимкнути повністю» + перевірка в runtime/engine.
- **Taras** UI + tauri-команди; **Den** мінімальний expose. Knox може паралельно (окремий файл-модалка).

## Фаза B6 — Мовна пара (найнижчий пріоритет)

- Вибір мовної пари (зараз лише uk-en; фундамент закладено — `LanguagePair` enum, `primary_langid_for_id`).
- **Bruno (data):** переконатися, що дані (layouts/LM/dict) вантажаться параметрично під пару.
- **Den (core):** мовно-агностичний шлях (вже багато є — `LanguageProfile` список).
- **Pylyp:** layout-switch для довільної пари + «cycle to next layout».
- **Taras:** UI-селектор пари.
- Робимо В ОСТАННЮ — спершу uk-en має бути ідеальним.
