# typofix-platform-virtual — правила

Реалізація `trait Platform` **повністю в пам'яті** — це наш рушій автономного
E2E-тестування без реальної ОС.

- Тримає віртуальні: текстовий буфер + позицію курсору, активне вікно
  (`WindowInfo`), поточну розкладку, чергу вхідних `InputEvent`.
- `apply(Action)` змінює віртуальний буфер так, як зробила б ОС: `TypeUnicode`
  вставляє текст у курсор, `DeleteChars(n)` стирає n перед курсором, `SwitchLayout`
  міняє поточну розкладку. Тести задають події й **читають буфер** — точно як
  побачив би користувач.
- API для тестів: enqueue подій + інспекція буфера/курсору/вікна.
- **Драйвер `drive()` не приймає `core::Context`** — інакше virtual залежав би
  від `core`, а `core` тримає virtual як dev-dep → цикл. Тому замикання отримує
  `(InputEvent, &WindowInfo, &LayoutId)`, а `Context` будує вже сам тест у `core`.
- Буфер — `Vec<char>`, курсор — індекс у **символах**, не байтах (Unicode-безпека
  `DeleteChars`/`TypeUnicode`).

**Готча (неочевидне й важливе):** virtual моделює НАШЕ уявлення про ОС, а не саму
ОС. Він НЕ ловить платформні баги: тайм-аут хука, гонку перемикання розкладки,
autocomplete-поля (де видимий текст ≠ кількість натисків), UIPI, auto-disable tap.
Їх ловить лише real-OS харнес (`docs/TESTING_STRATEGY.md` рівень 4). Не вважай
зелений virtual-тест доказом, що працює на реальній ОС.
