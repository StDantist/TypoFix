# Дизайн: перемикання розкладки НА ЛЬОТУ (mid-word live switch)

> Статус: **СПРОЄКТОВАНО, НЕ розпочато.** Чекає команди власника на старт.
> Дефолт: **вимкнено** (нова поведінка; eval її не бачить → вмикати лише після
> ручного живого калібрування). Пріоритет проєкту — precision-first.

## Мета
Перемикати розкладку ПОСЕРЕД слова, не чекаючи пробіл/Enter, щойно набрана
послідовність стає неможливою в поточній мові, але можливою в іншій
(`фв` → раннє перемикання на `ad`).

## Принцип (двосторонній гейт — тримає влучність)
На КОЖНОМУ натиску літери перевіряти ОБИДВІ умови разом:
1. **Поточна мова — глухий кут:** інтерпретація буфера в активній розкладці БІЛЬШЕ
   не є префіксом жодного реального слова (`!dict.has_prefix(current_text)`).
2. **Інша мова — живий префікс:** та сама послідовність клавіш в іншій розкладці —
   валідний префікс реального слова (`dict.has_prefix(other_text)`).
Лише коли ОБИДВІ → ранній перенабір+перемикання. Якщо обидві мови — глухий кут
(код/абревіатура) → НІЧОГО (чекати межу, як зараз). Симетрично UK↔EN.

## Ключове відкриття: FST уже дозволяє префіксні запити
`Dictionary` = обгортка над `fst::Set`. Префіксна перевірка БЕЗ нового артефакту й
без нової структури: сире проходження `set.as_fst()` — від `root()`, по UTF-8-байтах
`node.find_input(b)`; `None` → глухий кут, усі байти пройдено → живий префікс.
Zero-alloc, мікросекунди (вкладається у «<1мс на натиск»). Переюзаємо `uk.fst`/`en.fst`.
Готча: продзеркалити апостроф-нормалізацію + lowercase з `Dictionary::contains`.

## Точки змін
- `dict.rs` — новий `has_prefix(&self, prefix) -> bool` (+ юніти).
- `detector.rs` — нова ЧИСТА `live_decide(strokes, ctx) -> Option<Decision>`
  (узагальнює наявний `starts_impossible_uk` до префіксного глухого кута); нові поля
  `DetectorConfig`: `live_switch_enabled` (default `false`), `live_min_len` (default `4`).
- `engine.rs` — інтеграція в гілку `Class::Word` (успадковує ВЕСЬ апстрім-bypass:
  secure/виключення/синтетика/auto-repeat/Backspace); коротке замикання
  `handle_boundary` при піні.
- `buffer.rs` — нове поле `WordBuffer::live_locked` (скидається в `reset`/`invalidate`).
- `src-tauri` (Taras) — `BehaviorDto.live_switch`, мапінг у `detector_config_from`,
  bump `SCHEMA_VERSION`, картка «Поведінка» + i18n.

## Мід-ворд перенабір + когерентність буфера (НАЙСКЛАДНІШЕ)
Через `replacer::plan(&decision, None)` (separator=None, як `force_switch_last`):
`DeleteChars(current_text.chars().count())` + `SwitchLayout(other)` + `TypeUnicode(other_text)`.
**Буфер НЕ скидаємо** — лишаємо ті самі layout-незалежні `KeyStroke`; ОС тепер у новій
розкладці друкує наступні клавіші правильно. На межі: `live_locked==true` →
`handle_boundary` робить `reset()` і повертає порожньо (екран уже коректний; без
подвоєння). Один свіч на слово (пін гасить і live, і boundary далі).

## Undo для live (продуктове рішення — узгодити з власником)
Рекомендовано **варіант Б**: НЕ виставляти `pending_retype` на live (щоб не завчати
ФРАГМЕНТ слова в learned-винятки). Backspace тоді просто `pop`-не страйк; ескейп —
ручне перемикання/`revert`. Альтернатива А (повний pending) дає миттєвий Backspace-undo,
але забруднює learned фрагментами.

## Тести
- Юніти: `dict::has_prefix`; `detector::live_decide` (фв→Some(ad); обидва-глухий-кут→None;
  живий поточний префікс→None; veto→None; коротше за `live_min_len`→None; прапорець off→None).
- E2E virtual (`tests/live_switch.rs`): раннє перемикання серед слова; продовження слова
  дає коректний фінал без повторних дій; обидва-глухий-кут→нема дії; коректний ввід у
  власній мові не смикається; secure/виключення/veto не чіпаються; симетрія EN→UK.
- ⚠️ **Дірка автономії:** eval-харнес годує цілі слова в `decide`, оминаючи `step`/буфер
  → live НЕ бачить, precision-гарантії на нього НЕ дає. Покриття — лише юніти+virtual.
  Зафіксувати в реєстрі ПОЛІТИКИ ПЕРЕМИКАННЯ.

## Етапність (контроль ризику)
1. `has_prefix` + юніти (нуль впливу).
2. `live_decide` + поля config (default off; движок ще не кличе).
3. **Інтеграція + мід-ворд перенабір + когерентність (НАЙРИЗИКОВАНІШЕ для precision).**
4. Прапорець + UI (src-tauri).

## Ризики (чесно)
1. **#1 — діри покриття словника.** Реальний укр. префікс, відсутній у `uk.fst`
   (власні назви/рідкісні форми/сленг/неологізми), хибно вважається глухим кутом →
   перемикання посеред активного набору (гірше за boundary-помилку). Мітигатори:
   `live_min_len>=4` (дефолт, калібровано), обов'язкова умова «інша мова — живий
   dict-префікс», `recognizes` (user.txt стає важливішим запобіжником), пін одного
   свічу. → дефолт OFF + ручна калібровка.
   **Калібровка** (`typofix-data --bin live_calibrate`, реальні моделі): на in-dict
   словах FP≈0 (гейт коректний); на held-out OpenSubtitles FP падає 0.26%→0.03% при
   `live_min_len` 3→4, recall лишається 99.7% (лов на 1 страйк пізніше) → дефолт
   піднято до **4**. Held-out набір засмічений рос. словами (`еще`/`где` = ~66%
   FP-токенів) → реальна укр. FP-частота ще нижча. `user.txt` — точковий важіль на діри.
2. **#2 — ОС-гонка розкладки, ГОСТРІША мід-слова** (немає паузи-пробілу; швидкий друкар
   може натиснути наступну клавішу до застосування `SwitchLayout` → екран
   десинхронізується). Платформний ризик поза core; ловиться лише живим прогоном
   (`examples/live_engine.rs`), virtual не ловить. Можливий мітигатор у платформі (coalescing).
3. fst-префікси підтверджено доступні; апостроф/регістр — дзеркалити `contains`.
4. learned/veto keyed на повні слова → мід-ворд префікс може оминути learned-veto (низько,
   гаситься піном/варіантом Б).
