# fixtures/ — правила (ПРИВАТНІСТЬ — критично)

Golden-фікстури записаних сесій натисків (`*.jsonl`) для regression-тестів
(record/replay через `typofix-replay`).

**Залізне правило: ТІЛЬКИ синтетичні або скрабовані дані. Нуль реальних секретів.**
TypoFix бачить увесь ввід, тож реальний запис міг би містити паролі/токени. Такі
дані в репозиторій НЕ потрапляють — інакше ми зливаємо секрети в git назавжди.

- `*.expected.jsonl` (очікуваний вихід) **розмічається незалежно** — людиною або
  окремим інструментом, а НЕ генерується поточним движком. Інакше тест циклічний:
  ловить лише зміни, а не відхилення від задуму.
- Кожен новий баг → нова фікстура («баг як тест»).
- Пара файлів: `<name>.jsonl` (сесія: рядок `setup`, далі `event`-и) і
  `<name>.expected.jsonl` (рядки `text`/`action`). Приклад:
  `example_ghbdsn_uk.{jsonl,expected.jsonl}` (синтетика, `ghbdsn`→`привіт`).
- Деталі формату: `crates/typofix-replay` (docstring крейту). Стратегія:
  `docs/TESTING_STRATEGY.md` §2.
